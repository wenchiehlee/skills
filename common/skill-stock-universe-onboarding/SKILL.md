# Stock Universe Onboarding Skill

Use this skill to add stocks at the upstream source-list layer, not by hand-editing downstream generated outputs.

## What This Skill Means

There are three distinct onboarding targets:

1. **ConceptStocks new stock/ticker**: add a concept column to `../ConceptStocks/raw_companyinfo.csv`, add matching metadata in `../ConceptStocks/raw_conceptstock_company_metadata.csv`, and mark Taiwan companies exposed to that concept with `1` in the new column. This creates the ticker universe used by ConceptStocks scripts; it does not by itself guarantee quarterly income or segment rows exist.
2. **TAIEX monitor list stock**: add a Taiwan stock `代號,名稱` row to `../Selenium-Actions.Auction/觀察名單.csv`. This is the source for broad Taiwan monitoring lists such as `StockID_TWSE_TPEX.csv` in downstream repos.
3. **TAIEX focus list stock**: add a Taiwan stock `代號,名稱` row to `../Selenium-Actions.Auction/專注名單.csv`. This is the source for focus lists such as `StockID_TWSE_TPEX_focus.csv` in downstream repos.

Do not confuse these layers. A foreign ticker such as `000660.KS`, `005930.KS`, `MU`, or `NVDA` belongs in ConceptStocks metadata/concept columns, not in Taiwan monitor/focus CSVs. A Taiwan stock such as `2395` or `2382` belongs in monitor/focus CSVs and may also be marked as exposed to a ConceptStocks concept column.

## Source Files

Default paths, relative to a working repo such as `biztrends.TW`:

- Concept universe: `../ConceptStocks/raw_companyinfo.csv`
- Concept metadata: `../ConceptStocks/raw_conceptstock_company_metadata.csv`
- TAIEX monitor list: `../Selenium-Actions.Auction/觀察名單.csv`
- TAIEX focus list: `../Selenium-Actions.Auction/專注名單.csv`

Generated downstream files such as `data/ConceptStocks/raw_conceptstock_company_income.csv`, `data/ConceptStocks/raw_conceptstock_company_quarterly_segments.csv`, `data/cycle_mapping.csv`, `StockID_TWSE_TPEX.csv`, and `StockID_TWSE_TPEX_focus.csv` should be regenerated by their owning pipelines after source-list updates.

## Standard Workflow

1. Inspect current state before editing:

```bash
git -C ../ConceptStocks status --short
git -C ../Selenium-Actions.Auction status --short
```

2. Decide which onboarding targets apply:

- ConceptStocks only: foreign or US/Korea/global ticker concept, plus Taiwan company exposure markers.
- TAIEX monitor only: Taiwan stock should enter broad monitoring.
- TAIEX focus only: Taiwan stock should enter focused monitoring/reporting.
- Combined: Taiwan stock goes into monitor/focus and is also marked under an existing or new ConceptStocks concept column.

3. Use the helper script where possible:

```bash
python3 ../skills/common/skill-stock-universe-onboarding/scripts/add_stock_universe.py \
  --concept-field 'SK海力士概念' \
  --concept-ticker '000660.KS' \
  --concept-company 'SK hynix Inc.' \
  --concept-segments 'HBM,DRAM,AI memory' \
  --concept-exposed '2308,2317,2324,2357,4938' \
  --monitor-stock '6414,樺漢' \
  --focus-stock '2395,研華'
```

All arguments are optional, but each operation needs its own required fields:

- Concept metadata needs `--concept-field`, `--concept-ticker`, and `--concept-company`.
- Concept exposure needs `--concept-field` plus `--concept-exposed` stock IDs.
- Monitor/focus additions need `代號,名稱` pairs via `--monitor-stock` or `--focus-stock`.

4. Verify the exact rows/columns changed:

```bash
git -C ../ConceptStocks diff -- raw_companyinfo.csv raw_conceptstock_company_metadata.csv
git -C ../Selenium-Actions.Auction diff -- 觀察名單.csv 專注名單.csv
```

5. Run downstream regeneration only when the user asks or when the repo workflow requires it. Typical next steps are:

```bash
python3 ../ConceptStocks/scripts/update_conceptstocks.py --sync-concepts --out-dir ../ConceptStocks
python3 ../ConceptStocks/scripts/update_conceptstocks.py --all --out-dir ../ConceptStocks
python3 ../ConceptStocks/scripts/update_company_financials.py --ticker <TICKER> --out-dir ../ConceptStocks
python3 ../ConceptStocks/scripts/generate_quarterly_segments.py --out-dir ../ConceptStocks
```

For TAIEX monitor/focus consumers, run the owning repo's list refresh workflow, for example GoogleAlertManager `uv run python cli.py update-list` or similar project-specific sync commands.

## ConceptStocks Notes

- `raw_companyinfo.csv` is a Taiwan company table. It should not get a new row for `000660.KS` or `005930.KS`; it gets a new `*概念` column and Taiwan exposure markers.
- `raw_conceptstock_company_metadata.csv` maps that concept column to the foreign/global ticker.
- `update_conceptstocks.py` reads `*概念` headers from `raw_companyinfo.csv`, then maps those headers through metadata or hard-coded mappings to active tickers.
- Quarterly income/segment support may still require code changes in `../ConceptStocks/scripts/update_company_financials.py`, `../ConceptStocks/scripts/generate_quarterly_segments.py`, or provider-specific clients. Do not claim Revenue/YoY/Profit/GM coverage exists until rows are present in the generated income and segment CSVs.

## Data Rules

- Preserve CSV encoding as UTF-8 with BOM when the source file already has it.
- Preserve original column order; new concept columns should be inserted before `相關集團` when that column exists, otherwise before timestamp columns.
- For existing rows, write `1` only for explicitly exposed Taiwan stock IDs and `0` otherwise.
- Do not create duplicate stock rows. If a stock already exists with a different name, report the mismatch instead of silently changing it.
- Do not remove rows or columns unless the user explicitly asks.
- After editing upstream repos, check their individual git status. Commits must be made in the repo that owns the changed files.
